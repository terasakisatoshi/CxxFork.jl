## Per-Feature Details

### README build and load path

- **Role:** Julia package user on macOS arm64 with Julia 1.12.6 and current Apple developer tools.
- **Use Case:** Build and load CxxFork.jl exactly as the README recommends.
- **What they tried:** Ran `env JULIA_PKG_PRECOMPILE_AUTO=0 julia --project=. -e 'using Pkg; Pkg.build()'`, then `julia --project=. --compiled-modules=no -e 'using Cxx'`.
- **Discoverability:** The README clearly describes the supported Julia version, primary platform, build command, and smoke load command.
- **Setup:** `Pkg.build()` completed successfully.
- **Functionality:** Default `using Cxx` failed during Cxx initialization while parsing macOS SDK libc++ headers.
- **Expected vs Actual Outcome:** Expected a successful load. Actual result was a Clang parse failure in `/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk/usr/include/c++/v1`, including undeclared `__builtin_ctzg` and `__builtin_clzg`.
- **Blocked steps:** The default quick example could not be run after the default load failure.
- **Friction points:** The README does not mention that newer macOS SDK libc++ headers can be incompatible with the embedded Clang path.
- **Doc suggestions:** Document a known-good macOS SDK/header configuration or add a troubleshooting note for `__builtin_ctzg` / `__builtin_clzg` load failures.

### README quick C++ FFI example

- **Role:** Julia package user trying the first documented C++ interop example.
- **Use Case:** Define a simple C++ function through `cxx"""..."""`, call it with `@cxx`, and evaluate an inline expression with `icxx`.
- **What they tried:** After the default load failed, reran the smoke path with `CXXJL_HEADER_DIRS` pinned to the installed macOS 15.4 SDK headers and `CXXJL_NOSTDCXX=1`, then ran the README quick example.
- **Discoverability:** The README quick example is easy to find and self-contained.
- **Setup:** The workaround load succeeded with a non-fatal `"Unsupported compiler detected"` warning from the macOS 15.4 SDK headers.
- **Functionality:** The quick example passed under the pinned SDK header environment: `@cxx(cxx_smoke_add(41)) == 42` and `icxx"1 + 2;" == 3`.
- **Expected vs Actual Outcome:** The feature works once the header incompatibility is avoided, but this is not the default documented path.
- **Blocked steps:** None after the SDK header workaround.
- **Friction points:** The successful workaround is not discoverable from README.
- **Doc suggestions:** If macOS 15.x SDK headers are an intended workaround, document the exact `CXXJL_HEADER_DIRS` shape and explain that current macOS 26.x SDK headers may fail.

## Expected vs Actual Outcome

The expected end-to-end README path was not achieved with default environment settings. `Pkg.build()` succeeded, but `using Cxx` failed before user code could run. A diagnostic rerun showed that selecting the older installed macOS 15.4 SDK headers makes `using Cxx` and the quick example pass, which narrows the failure to default macOS SDK header selection and compatibility.

## Issues Found

1. **Critical: Default macOS arm64 load path fails with current SDK headers.**
   - Environment: Julia 1.12.6, macOS arm64, LLVM 18.1.7, Xcode 26.4.1 / Apple clang 21.0.0, Command Line Tools SDK symlink `MacOSX.sdk -> MacOSX26.4.sdk`.
   - `deps/path.jl` selected `/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk/usr/include/c++/v1`.
   - `using Cxx` failed because Cxx's embedded Clang path did not recognize libc++ header builtins `__builtin_ctzg` and `__builtin_clzg`.

2. **Documentation mismatch: generated docs still contain legacy installation/platform claims.**
   - `docs/src/index.md` says the package has out-of-box installation on all 64-bit Julia 1.1 Tier 1 platforms and recommends `pkg> add Cxx`.
   - README says Julia 1.12 is the baseline, macOS arm64 is primary, Linux is smoke-only, and Windows is deferred.

3. **Documentation mismatch: REPL docs describe an available experimental C++ REPL without the README's default-off caveat.**
   - README says the legacy C++ REPL pane is disabled by default.
   - `docs/src/repl.md` describes entering C++ mode from the Julia REPL without mentioning `CXX_ENABLE_REPL=1`.

## Suggestions

1. Adjust macOS header selection to avoid defaulting to SDK headers that the embedded Clang path cannot parse, or add a compatibility gate with a clear error and workaround.
2. Add README troubleshooting for `__builtin_ctzg` / `__builtin_clzg` failures, including a known-good SDK/header workaround if that is supported.
3. Update `docs/src/index.md` to match the README's Julia 1.12 and platform-support language.
4. Update `docs/src/repl.md` to state that REPL integration is experimental and requires `CXX_ENABLE_REPL=1`.
