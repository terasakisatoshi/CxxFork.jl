## Per-Feature Details

### Historical `Using Cxx.jl` examples

- **Role:** Julia 1.12 user evaluating whether old Cxx.jl examples still work with CxxFork.jl.
- **Use Case:** Run the examples from `https://github.com/JuliaInterop/Cxx.jl#using-cxxjl` against the current checkout.
- **What they tried:** Ran `Pkg.build()`, loaded `Cxx`, then executed the examples for simple embedded C++ functions, numeric and string arguments, Julia expression splicing, inline `icxx`, enums, C++ classes, and the `ArrayMaker` shared library flow.
- **Discoverability:** The upstream examples are easy to find. The fork README already has a smaller `Quick Example` and a `Legacy Examples` note, but it does not spell out that several upstream examples are currently unsupported.
- **Setup:** `env JULIA_PKG_PRECOMPILE_AUTO=0 julia --project=. -e 'using Pkg; Pkg.build()'` succeeded. `julia --project=. --compiled-modules=no -e 'using Cxx'` succeeded with the existing non-fatal macOS SDK header warning.
- **Functionality:** Examples 1-3 worked. Example 5 worked when supplied standard input. Example 6 worked, with modern display type names under `Cxx.CxxCore`. Example 4 failed. Examples 7 and 8 failed on member function calls.
- **Expected vs Actual Outcome:** The historical section is only partially supported. The full sample set cannot be added to README as supported documentation.
- **Blocked steps:** None were skipped for safety. The interactive input example was run with piped input. The shared-library example built the test library in a Julia temporary directory.
- **Friction points:** Failures are low-level Julia/Cxx internals errors, not user-facing unsupported diagnostics.
- **Doc suggestions:** Keep README examples limited to currently verified smoke examples, and track upstream example parity in an issue until Julia expression splicing and member function calls are fixed or explicitly declared unsupported.

## Expected vs Actual Outcome

Expected: the old `Using Cxx.jl` samples run on the modern Julia 1.12 baseline.

Actual:

- `cxx""" ... """` plus `@cxx mycppfunction()` works.
- `@cxx printme(jnum)` works for numeric arguments.
- `@cxx printme(pointer("John"))` works for string pointer arguments.
- Julia expression splicing inside `cxx"""..."""`, such as `$:(println(...)::Nothing)` and the modernized `$:(println(...)::Cvoid)`, fails with `FieldError: type Core.TypeName has no field mt`.
- `icxx"""..."""` with standard input worked when input was provided.
- C++ enum access and static enum-returning calls worked, although display names now include `Cxx.CxxCore`.
- `@cxxnew Hello()` succeeds, but `@cxx hello_class->hello_world(pointer(tstamp))` fails.
- The `ArrayMaker` shared library can be compiled, loaded, included, and constructed, but `@cxx maker->fillArr()` fails.

## Issues Found

1. **Critical: Julia expression splicing in `cxx"""..."""` is broken on Julia 1.12.**
   - Reproduction: define `void testJuliaPrint() { $:(println("hello")::Cvoid); }`.
   - Failure: `FieldError: type Core.TypeName has no field mt`.
   - Evidence points to `src/typetranslation.jl`, where callable Julia type translation still accesses `T.name.mt`, a field not present in Julia 1.12 `Core.TypeName`.

2. **Critical: `@cxx obj->method(...)` member function calls fail for simple user-defined classes.**
   - Reproduction: construct a simple class with `@cxxnew`, then call a non-template member method.
   - Failure includes Clang diagnostics like `'hello_world' following the 'template' keyword does not refer to a template`, followed by `UndefVarError: name not defined in Cxx.CxxCore`.
   - The same failure blocks both the class example and the shared-library `ArrayMaker` example.

3. **Documentation gap: README says most historical usage still applies, including member calls, but that is broader than the verified Julia 1.12 behavior.**
   - The current README's quick example is accurate, but the `Legacy Examples` wording may imply support for old examples that are not currently supported.

## Suggestions

1. File an issue tracking historical `Using Cxx.jl` example parity before adding those samples to README.
2. Fix or explicitly gate Julia expression splicing in `cxx"""..."""` for Julia 1.12.
3. Fix or explicitly gate `@cxx obj->method(...)` member function calls for user-defined classes.
4. After fixes, add a focused README section with only verified examples and include a test that exercises the documented snippets.
